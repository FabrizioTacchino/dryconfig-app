# F20.6 — Bulk update saltava labor finitura con BOM vuoto

**Data risoluzione**: 2026-05-13
**Severità**: 🔴 critica (era la causa residua del bug "Aggiorna prezzi cambia")
**Commit**: `fb692b3`

## Sintomo

Dopo F19 + F20 (1-5), il bug "Aggiorna prezzi cambia il prezzo" persisteva. L'utente vedeva delta di ~€5 dopo il click.

## Diagnosi via DB

Query sull'estimate_stratigraphy reale:
```
finish_level = 'Q2'
finish_components_data = []
finish_labor_minutes_per_sqm = 10.00
unit_cost = 13.41
```

Q2 di default ha **0 componenti BOM** ma **10 min/m² di labor**.

## Root cause

In `useBulkUpdateEstimateStratigraphyPrices.tsx`:

```ts
if (Array.isArray(finishSnap) && finishSnap.length > 0) {
  // calcola materialsCost + laborCost
}
// ELSE: newFinishCost stays at 0 — labor IGNORATA
```

In `useCreateEstimateStratigraphy.fetchFinishCostSnapshot` invece:

```ts
// Iterate components (no-op se vuoto)
// SEMPRE calcola laborCost
const laborCost = (laborMin * hourlyRate) / 60;
return { finishCostPerSqm: materialsCost + laborCost, ... };
```

Quindi:
- Create: `unit_cost = comprehensive + (0 + 10×30/60) = comprehensive + 5.00`
- Aggiorna prezzi: `unit_cost = comprehensive + 0 = comprehensive` (perde 5.00)

Stesso identico delta visto dall'utente.

## Fix

Condizione bulk ora:
```ts
const hasComponents = Array.isArray(finishSnap) && finishSnap.length > 0;
if (hasComponents || (hasFinishLevel && (newFinishLaborMin ?? 0) > 0)) {
  // calcola materialsCost (solo se hasComponents)
  // SEMPRE somma laborCost
}
```

`materialsCost` resta 0 se non ci sono componenti, ma la labor viene sempre inclusa.

## Lezione

- Quando una formula ha **2 input indipendenti** (materiali + labor), guardia di skip su uno solo dei due è bug pattern. Le guardie devono o coprire entrambi o nessuno.
- Sempre dare il primo sguardo ai **dati reali del DB** quando un bug "calc-vs-calc" persiste. La differenza esatta (qui €5) tipicamente smaschera quale termine viene perso.
- Anche un calcolo "nullo" (laborCost = 0×30/60 = 0) non è un valore "vuoto" che giustifica skip — è un valore valido che va sommato per coerenza.
